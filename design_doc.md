# 025_クマ出没情報マップPJ 設計書 (Design Doc)

## 1. プロジェクト概要
北海道内のクマ出没情報を可視化し、住民の安全を守るWebアプリケーション。

## 2. 開発フェーズ

- [/] フェーズ2：UI/UXデザイナー (進行中)
- [ ] フェーズ3：データ設計・システムアーキテクト
- [ ] フェーズ4：プログラマー（実装）

## 3. 要件定義 (フェーズ1合意事項)
- **対象エリア**: 北海道
- **基盤**: Webアプリ (Vite + Leaflet.js)
- **コア機能**: 
    - クマ出没情報のマップ表示（日時、出典URL含む）
    - 注意エリア（出没地周辺）の可視化
    - 現在地連動の接近アラート

## 4. UI/UX設計 (フェーズ2案)

### 4.1. カスタマージャーニー (画面遷移)
1. **初期ロード**: GPS使用許可を求め、現在地を中心にマップを展開。
2. **メイン画面**: 最新の出没ピンと「注意エリア（オレンジ色の円）」が表示されたフルスクリーンマップ。
3. **詳細確認**: ピンをタップすると画面下部から「詳細カード（ボトムシート）」がスライドアップ。出没日時、状況、出典元へのリンクを表示。
4. **アラート**: ユーザーが注意エリアに侵入すると、画面上部に赤い警告バーが表示され、デバイスがバイブレーション（可能な場合）で通知。

### 4.2. UIデザインのポイント
- **配色**: 視認性の高いダークモードを基調とし、注意エリアはオレンジ、極めて新しい情報は赤で表現。
- **直感性**: 「今、自分が危ないかどうか」が1秒でわかるよう、画面中央の自車位置（自歩位置）アイコンの状態を変化させる。
- **フィルター**: 過去24時間、3日間、1週間などの期間切り替えスイッチを配置。

## 5. 技術スタック
- フロントエンド: React / Vite
- マップ: Leaflet.js
- バックエンド（データ取得）: Python (Scraping)
- ホスティング: Vercel / GitHub Pages
